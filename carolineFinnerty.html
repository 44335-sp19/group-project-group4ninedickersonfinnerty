Programming 2 RPG Project

Completed in the Computer Programming 2 course, the RPG project required that there be 5 different player classes, 5 different types of monsters, and 5 scenes. To help keep the overall code short and sweet, methods were created to take care of the repetative task in RPG's such as battles, and user prompts. At the start of each turn the user would be prompted with the question of which direction they would like to move next. The user would respond with either North, South, East or West. If the player entered an invalid command the program would call them silly and ask that they try again.
After picking a direction they would either happen appon a scene or start a monster encounter. The scene would just tell the character what they saw, and then move into having them pick another direction. The monster encounter would ask the player if they would like to fight or try to run. If the player tried to run, the program would "roll a dice" and determine if the player is able to get away. If the player is unlucky they would take damage and lose the encounter. If the player chooses to fight, the program would also "roll a dice" to determine if the player is succesful. Winning in a monster fight means your character earned points, losing means you lost health points. The game would end when the user either quits or loses all their health points.
